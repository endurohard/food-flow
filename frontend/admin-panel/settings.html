Настройки телефонии PJSIP

Статус: Не подключено
Тип подключения
Выберите тип транспорта для подключения к SIP серверу
Параметры SIP сервера (UDP/TCP)
IP адрес или доменное имя SIP сервера
По умолчанию 5060
Протокол для SIP соединения
Параметры WebSocket
Полный URL WebSocket сервера (включая протокол ws:// или wss://)
Примеры WebSocket URL:
  • wss://it005.ru:8089/ws - безопасное соединение (рекомендуется)
  • ws://192.168.1.100:8088/ws - обычное соединение
  • Для Asterisk обычно используется порт 8088 или 8089
⚠️ Важно для WebSocket:

Для WebSocket подключения используется другой пароль. Убедитесь что в настройках пользователей указан правильный пароль для WebSocket (поле "WS Пароль").

Порты для медиа (RTP)
ℹ️ Когда нужны RTP порты:
  • Native PJSIP (UDP/TCP/TLS) - ✅ Необходимы. Сервис обрабатывает аудио напрямую.
  • WebSocket (WS/WSS) - ❌ Не нужны. Медиа обрабатывает браузер через WebRTC.
  • Только AMI - ❌ Не нужны. AMI только управляет звонками без медиа.
Важно! Убедитесь что на вашем оборудовании открыты:
  • Порт 5060 (или указанный выше) для SIP - протоколы TCP и UDP
  • Диапазон портов 5700-5750 для RTP - протокол UDP
  • Правила применяются для входящего и исходящего трафика

Настройка доступа сотрудников к телефонии

Укажите логин и пароли АТС для каждого сотрудника, который будет пользоваться телефонией.

Типы паролей:
  • Пароль SIP - используется для нативного подключения (UDP/TCP/TLS)
  • Пароль WS - используется для WebSocket подключения (может отличаться)
